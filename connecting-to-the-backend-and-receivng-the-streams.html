Connecting to the backend and receivng the streams

A short overview how to connect the sdk and receive the current liste streams.

Getting started

After you started the SDK and received the MycrocastRaydioSDK

You can start the connect()function.

/// connect the SDK, returns true if successfull
func connect() async -> Bool

But before you do you should register a delegate for the RaydioSDKStateDelegateto receive updates for the state of the sdk. Or use the combine publisher.

Both are found in the SDKStateAccess from sdkStateAccess

public protocol RaydioSDKStateDelegate {
    func onRaydioSDKStateChanged(_ state: RaydioSDKState)
}

public enum RaydioSDKState: Equatable {
    case idle
    case connected
    case disconnected
}

Use the RaydioSDKProviding/sdkStateDelegate to register the delegate.

After you registered the delegate and called the connect function you will receive a state update with either connected or disconnected.

Handling disconnected

Everytime you receive the disconnected state you should reconnect the sdk if possible by calling the RaydioSDKProviding/reconnect() method.

When do you receive the disconnect state?

We internally use websockets, so whenever the socket is closed. This can happen for different reasons:

  • The app goes into the background

  • The network connection is lost

  • A problem establishing the connection to our backend

You need to observe on you own the network and if the app is in foreground/background. See

If the app is in background you need to wait until it moved to the foreground to reconnect.

If the network was lost, you need to wait until the network was regained.

Both are further described here

Handling connected

You now have established the connection to our backend and can start requesting the current live streams:

For that call the refreshStreams() function from the sdk.

Before doing this you should register to receive updates.

Everything related to streams is handled via the StreamsAccess from streamsAccess

/// The interface to access everything related to the streams
public protocol StreamsAccess: AnyObject {
    /// The list of currently available streams
    var streams: [MycrocastStream] {get}

    /// Publisher that contains the current list and will
    /// publish the list again anytime an update happens either add, update or remove
    var streams$: AnyPublisher<[MycrocastStream], Never> {get}

    /// Use this publisher in case you want to react to an add event
    /// as this will always publish the lattest added stream
    /// This does not hold state inernally (Passthrough)
    var streamAdded$: AnyPublisher<MycrocastStream, Never> {get}

    /// Use this publisher in case you want to react to an remove event
    /// as this will always publish the lattest removed stream
    /// This does not hold state inernally (Passthrough)
    var streamRemoved$: AnyPublisher<MycrocastStream, Never> {get}

    /// Use this publisher in case you want to react to an update event
    /// as this will always publish the lattest updated stream
    /// This does not hold state inernally (Passthrough)
    var streamUpdated$: AnyPublisher<MycrocastStream, Never> {get}

    /// You can register a delegate here in case you do not want to use
    /// Combine that offers the same functionality but with callbacks
    var streamsDelegate: StreamsDelegate? {get set}
}

Either use the publishers or register the delegate.

The delegate provides the same functionality and has the following strucutre:

public protocol StreamsDelegate {

    /// Will be called each time a new stream is added
    /// with the stream that was just added
    func onStreamAdded(_ stream: MycrocastStream)

    /// Will be called each time a stream is removed
    /// with the stream that was just removed
    func onStreamRemoved(_ stream: MycrocastStream)

    /// Will be called each time a stream is updated
    /// with the stream that was just updated
    func onStreamUpdated(_ stream: MycrocastStream)

    /// Will be called each time the stream list is updated with the new
    /// List. This is triggered for each create/update/delete
    func onStreamListChanged(_ streams: [MycrocastStream])
}

MycrocastStream

A MycrocastStream is a single broadcast from a user via MycrocastStudio


/// A single stream that is currently online and playable
///
public struct MycrocastStream: Encodable {

    /// The language of the stream as the native name
    /// e.g for German it would state: Deutsch
    public var language: String
    // The id of the stream
    // this is used for playing and others
    public var streamId: String
    // is the stream currently muted from the streamer side
    public var isMuted: Bool
    // The match id for the game provided by the backend
    public var matchId: String

    // The title provided by the streamer
    public var title: String
    // The streamer seems to have connection issues
    public var connectionIssues: Bool
}

When to request streams

After the intial connection was established you should request the streams once.

The delegate function onStreamListChanged(_:) will be called with all currently available groups. The same hold true for the publisher.

After that you will receive live updates on the other 3 functions of the delegate as long as the sdk is connected.

This means that you should request streams again when the sdk is potencially coming from a disconnected state.

So when moving the app to the foreground or when network connection has been reastablished.

This can always be done in the state change listener, as the sdk changed back to state connected if you call the appropriate functions.
