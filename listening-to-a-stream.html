Listening to a stream

You app should have the background capability audio enabled

and you should configure the audio session:

  private func configureAudioSession() {
        do {
            try AVAudioSession.sharedInstance().setCategory(.playback)
            try AVAudioSession.sharedInstance().setActive(true)
        } catch {
        }
    }

After receiving the streams you can start listen to a stream by providing the corresponding streamId of the MycrocastStream to the playStream(_:) function of the sdk.

func playStream(_ streamId: String) async -> ListenFailedReason?

This function is async and should be awaited.

The returned ListenFailedReason informs if anything went wrong. If it is not present the play was successful.

Everything concerning the actual listen state is provided by the the

listenStateAccess

/// This protocol provides anything that belongs to the listening state
public protocol ListenStateAccess: AnyObject {
    /// Publisher that pushes the latest play state.
    /// The initial value is .idle
    var listenState$: AnyPublisher<RaydioPlayState, Never> {get}

    /// The delegate to register callbacks for the play state change
    /// Use this or the reactive approach with the listenState$
    var listenStateDelegate: ListenStateDelegate? {get set}

    /// determine if you are currently listening to the provided streamToken
    func isListeningTo(_ streamToken: String) -> Bool
}

To receive updates of the play state you first should register the corresponding delegate to receive state updates. This needs to be done on ListenStateAccess

public protocol ListenStateDelegate {
    func onListenStageChanged(_ playState: RaydioPlayState)
}

public enum RaydioPlayState: Equatable {
    case playing // we are currently playing
    case idle // initial state
    case connecting // in process of connecting
    case failed(error: RaydioError) // something might went wrong
    case paused // was connected, now paused
    case disconnected // not connected anymore
}

A description of RaydioError can be found here

This needs to be done only once.

Alternatively you can also subscribe to the publishers provided by the ListenStateAccess

    /// Publisher that pushes the latest play state.
    /// The initial value is .idle
    var listenState$: AnyPublisher<RaydioPlayState, Never> {get}

The disconnected state

The disconnected state can be normal in case the listener paused. Otherwise this can be based on network issues of either the listener or streamer.

The failed state

The failed(error:) can follow the disconnected state and represents a broken audio connection where a automatic reconnection is not possible.

Most of the time this reprents that either for example the streamer stopped, has network issues or you can have network issues.

Remote Play and Control

The SDK internally does not use any player provided by apple (AVPlayer or any other of the AVAudio).

Therefore the developer needs to decide if the remote control elements should be displayed and how.

The usage of the MPNowPlayingInfoCenter is recommended and to update the nowPlayingInfo property when the stream is started and ended as well to use the MPRemoteCommandCenter and register for play and pause events.
